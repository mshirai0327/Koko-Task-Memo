## 1. コンセプトと設計哲学

### 核心思想

「**忘れてもいい**」を仕様にする。タスクは72時間後に自動削除され、管理しなくてよいことが前提。Jira・ClickUp のような重厚なタスク管理ではなく、付箋や頭の中のメモに近い感覚を目指す。

### 設計原則（優先順位順）

1. **触って楽しいこと** — CSSアニメーション豊か、インタラクションが気持ちいい
2. **シンプルであること** — 機能は最小限。設定画面はできるだけ作らない
3. **軽くてよいこと** — 忘れたタスクは消えてよい。完璧な記録は不要
4. **どこからでも使えること** — PC・スマホ両対応。ブラウザアクセス必須
5. **親しみがあること** — 日本語UI、柔らかいフォント、感情のあるコピー

### アンチパターン（やらないこと）

- タスクの優先度・ラベル・カテゴリ管理
- リマインダー・通知機能（将来的にはあってもいい）
- チームシェア・コラボレーション機能
- 複雑な設定UI
- タスクの無限保存・アーカイブ

## 2. 機能仕様

### 2-1. タスク追加

- テキスト入力 → Enterキー または「+」ボタンで追加
- 最大文字数: 120文字
- 追加時: バウンスアニメーション付きでカードがスライドイン
- 追加と同時に `createdAt` タイムスタンプを記録（TTL計算の基準）

### 2-2. タスク完了

- チェックボタンをクリックでトグル
- 完了状態: 打ち消し線・透明度低下・緑チェック
- 完了タスクは「おわった」セクションに移動
- 完了しても72時間TTLは継続（完了しても消える）

### 2-3. タスク編集

- タスクテキストをクリックで `contenteditable` 編集
- Enterキーまたはフォーカスアウトで確定
- 空文字での確定は不可（元のテキストに戻す）

### 2-4. タスク削除

- ✕ボタンクリックで vanish アニメーション → DOM削除
- TTL切れによる自動削除も同様のアニメーション

→より単純にするため、削除も完了も一緒にしたらどうか？　できるだけシンプルに、頭を使わせたくない。

### 2-5. TTL（72時間自動削除）

- 各タスクに `createdAt` を保持
- 残り時間をカード右端に表示（例: `48h 30m` / `2m 10s`）
- 残り時間30%以下で `--accent2`（ピンク）色に変化
- カード下部にTTLバー（残り時間比率で縮む）
- 毎分チェック。切れたものを localStorage から除去 + アニメーション削除

## 3. データ構造

```js
// localStorage キー
const STORAGE_KEY = 'kiemono_tasks_v1';

// タスクオブジェクト
{
  id: string,        // Date.now().toString(36) + random
  text: string,      // タスクテキスト（最大120文字）
  createdAt: number, // Unix ms タイムスタンプ
  done: boolean,     // 完了フラグ
}
```

- ストレージキーのバージョニング（v1, v2...）でマイグレーション対応
- 起動時に期限切れタスクをフィルタリングして保存

## 4. デザインシステム

### カラーパレット

| 変数 | 値 | 用途 |
|------|----|------|
| `--bg` | `#0F0E17` | 背景（ほぼ黒） |
| `--surface` | `#1A1828` | カード背景 |
| `--surface2` | `#242136` | ホバー時 |
| `--ink` | `#FFFFFE` | メインテキスト |
| `--ink-dim` | `#A7A5BC` | サブテキスト |
| `--ink-ghost` | `#4A4862` | ヒント・ラベル |
| `--accent` | `#FF8906` | オレンジ（メインアクション） |
| `--accent2` | `#E53170` | ピンク（緊急・削除） |
| `--accent3` | `#3DA9FC` | ブルー（TTLバー） |
| `--done` | `#2CB67D` | 完了（グリーン） |

### タイポグラフィ

| 変数 | フォント | 用途 |
|------|---------|------|
| `--font-hand` | Klee One | ロゴ・プレースホルダー・空状態テキスト |
| `--font-ja` | Zen Kaku Gothic New | 本文・タスクテキスト（weight 300/400/700） |
| `--font-mono` | DM Mono | TTLラベル・セクションヘッダー・フッター |

### レイアウト

- 最大幅 560px、中央寄せ
- padding: `48px 20px 120px`（フッターとの重なり回避）
- モバイル（≤480px）: ロゴ 1.8rem、上部パディング 32px

### アニメーション仕様

| 名前 | 内容 |
|------|------|
| `cardIn` | 追加時: Y軸-12px + scale(0.96) → 通常。`cubic-bezier(0.34,1.4,0.64,1)` バウンス |
| `vanish` | 削除時: opacity+scale縮小 → max-height:0 でスペースも消える。0.6s |
| `fadeSlideDown` | ヘッダー・入力欄の初期表示。0.6s ease |
| `float` | 空状態アイコンの浮遊。3s ease-in-out infinite |
| `drift` | 背景グロウの揺れ。20s infinite。3つの光が異なるdelay |
| TTLバー | width: CSSトランジション 1s linear で縮む |

## 5. 技術構成（現フェーズ）

| 項目 | 内容 |

### Phase C: ネイティブアプリ展開（コスト: 低）

| ブラウザ | PWA | そのまま継続 |

※ Tauri でモバイルをやらない: 2024年10月 GA でまだ Capacitor より未成熟  
→ `docs/phases/phase-c-native.md` を参照

### Phase D: 軽量ソーシャル機能（将来構想）

- localStorage の容量上限（通常5MB）
- `contenteditable` 編集はIMEで稀に二重入力が起きる可能性あり
- TTL更新が1分間隔なので残り時間表示が最大1分ずれる
